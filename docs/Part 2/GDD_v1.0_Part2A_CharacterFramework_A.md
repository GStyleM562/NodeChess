# Part 2A - Character Framework

---

# 1. Character Design Philosophy

Characters are the primary content unit of the game.

Characters should not simply be stronger versions of other characters.

Characters are designed around:

• Role
• Combat Identity
• Attack System
• Mobility
• Utility
• Rank Up Potential
• Passives
• Board Control


Characters should encourage different playstyles.

Examples:

Aggressive

Defensive

Control

Objective Rush

Energy Generation

Debuff

Support

Counterattack

Displacement

# 3. Character Classes


Character Classes are intended to help categorize gameplay.

Classes do NOT restrict abilities.


---

## Balanced


Characteristics:

Average movement.

Average combat.

Reliable.


Recommended stamina

2
